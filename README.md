# AutoTracking

Сервис для автоматической отметки посещаемости студентов в Google Sheets по QR-коду.

Каждый студент носит с собой QR-код со ссылкой вида `http://<сервер>:8090/<курс>?id=<studentID>`.
Камера на входе сканит код (`main.py`), дёргает эту ссылку GET-запросом — сервер (`main.go`)
находит нужную таблицу курса, находит колонку с сегодняшней датой и строку студента,
ставит `1` в ячейку и красит её зелёным.

## Структура проекта

```
main.go              — точка входа: поднимает логгер, инициализирует Attendance и Course, запускает HTTP-сервер
main.py              — отдельный скрипт: сканит QR через веб-камеру и дёргает ссылку из него
handler/handler.go   — HTTP-хендлер: разбирает URL (/<курс>?id=<id>) и вызывает Attendance.Track
Attendance/          — работа с Google Sheets API: поиск нужной ячейки по дате/id, запись, покраска
Course/               — курс -> spreadsheetID в памяти, грузится из config.env
Conf/                 — парсер config.env (формат КУРС=SpreadsheetID)
logger/               — общий логгер: пишет и в консоль, и в Server.log
credentials.json      — сервис-аккаунт Google (в .gitignore, в репозиторий не попадает)
config.env            — список курсов и их spreadsheetID (в .gitignore)
config.env.example    — пример формата config.env
```

## Формат таблицы в Google Sheets

Лист должен называться `Attendance`. Ожидаемая структура:
- строка 3 — даты (в любом формате, который Google Sheets хранит как serial number)
- начиная со строки 4, колонка A — ID студента, дальше по строке — отметки посещаемости по датам

## Настройка

1. Создай сервис-аккаунт в Google Cloud, включи Sheets API, скачай JSON-ключ и положи его в
   `credentials.json` в корне проекта. Дай сервис-аккаунту доступ на редактирование к нужным таблицам.
2. Скопируй `config.env.example` в `config.env` и пропиши свои курсы:
   ```
   МойКурс=1iDIl2sGg8nOBugNQfcSjmqDnq5lFuv2wL7wqQEKVTeU
   ```
   Значение — это ID таблицы (из её ссылки в Google Sheets).
3. `credentials.json` и `config.env` уже в `.gitignore` — секреты в git не попадают.

## Запуск

```bash
go build -o autotracking .
./autotracking
```

Сервер поднимается на порту `8090`. Логи пишутся одновременно в консоль и в `Server.log`
(файл создаётся/дополняется в рабочей директории).

## QR-сканер (main.py)

Отдельный вспомогательный скрипт на Python — сканит QR через веб-камеру и дёргает ссылку
из него GET-запросом, без открытия браузера. Пищит на успех/ошибку, показывает статус на весь экран.

```bash
pip install opencv-python requests --break-system-packages
python3 main.py
```

`q` — выход, `f` — переключить полноэкранный режим.

## Известные особенности / на что обратить внимание

- Специальный `id=1488` в handler'е ставит отметку "отсутствовал" вместо "присутствовал" —
  сейчас захардкожен прямо в коде. Стоит вынести в конфиг.
- `go.mod` и `go.sum` сейчас в `.gitignore` — обычно их коммитят для воспроизводимой сборки.
